\begin{convention}[Index suppression and aggregation from the root variable]
\label{conv:notation}

Every variable in the model is defined with a set of root indices. For example, the variable $\phi_{cijm}$ is defined for $(c \in C, i \in V, j \in V, m \in \mathcal{M})$; we call this the root variable. Writing a variable with fewer indices always denotes a derived variable: one or more indices have been suppressed and the root values aggregated according to the rules below.

\medskip
\noindent\textbf{Rule 1 (Sum aggregation).} 
Dropping indices from any variable denotes the sum of the root variable over every value of those indices:
\begin{align*}
  \phi_{cij} &\coloneqq \sum_{m \in \mathcal{M}} \phi_{cijm}, &
  \phi_{ijm} &\coloneqq \sum_{c \in C} \phi_{cijm}, &
  \phi &\coloneqq \sum_{c \in C} \sum_{i \in V} \sum_{j \in V} \sum_{m \in \mathcal{M}} \phi_{cijm}.
\end{align*}
For continuous and integer variables, this is the natural interpretation. For binary variables, the result is an \textit{integer count} of how many root-level binaries in the group equal 1; it is not itself binary. For example, $X_{cij} \coloneqq \sum_{m \in \mathcal{M}} X_{cijm}$ counts active modes for type $c$ on lane $(i,j)$ and takes values in $\{0, 1, \dots, |\mathcal{M}|\}$.

Sum aggregation is primarily a notational shorthand. However, we can also explicitly introduce derived variables to the model (e.g., $\phi_{cij}$) and define the associated equality constraint:
\begin{equation}
    \phi_{cij} = \sum_{m \in \mathcal{M}} \phi_{cijm} \quad \forall c \in C, (i,j) \in E \label{eq:aux-sum}
\end{equation}

\medskip
\noindent\textbf{Rule 2 (Dot notation for the node-pair $(i,j)$).} 
The root variable carries two active indices from the same set $V$. Suppressing one while keeping the other would be ambiguous, so a dot $(\,\cdot\,)$ marks the suppressed position:
\begin{align*}
  \phi_{c \cdot j m} &\coloneqq \sum_{i \in V} \phi_{cijm}, &
  \phi_{ci \cdot m} &\coloneqq \sum_{j \in V} \phi_{cijm}.
\end{align*}
Rule 1 applies as normal: the result is a sum or an integer count.

\medskip
\noindent\textbf{Rule 3 (OR-aggregated binaries).} 
A new binary decision variable ($\overline{X}$) is introduced to represent a logical \textit{OR}, linked to the sum aggregate $X_{cij}$ via:
\begin{align}
  \overline{X}_{cij} &\leq X_{cij} \label{eq:or-m-lb} \\
  X_{cij} &\leq |\mathcal{M}| \cdot \overline{X}_{cij} \label{eq:or-m-ub}
\end{align}
Together, these enforce $\overline{X}_{cij} = 1 \iff X_{cij} \geq 1$.

\medskip
\noindent\textbf{Rule 4 (At-most-one aggregation).} 
When at most one commodity type may be active on a lane, a new binary $\widehat{X}_{cij}$ is introduced and linked to the sum aggregate $X_{cij}$ by:
\begin{align}
  \widehat{X}_{cij} &\leq X_{cij} \label{eq:At-most-one-m-lb} \\
  X_{cij} &\leq   \widehat{X}_{cij} \label{eq:At-most-one-m-ub}
\end{align}
